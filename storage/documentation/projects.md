# Project Management

Manage projects, tasks, milestones, and resource allocation. Learn how to plan projects, track progress, manage budgets, and deliver successful outcomes for your clients.

## Understanding Projects

Projects in Nestogy represent discrete bodies of work with defined scopes, timelines, and deliverables:

- Infrastructure upgrades
- Cloud migrations
- Software implementations
- Office moves
- Security assessments
- Network installations

---

## Creating a Project

To create a new project:

1. Navigate to **Projects** in the sidebar
2. Click **Create Project**
3. Select the client
4. Configure project details:
   - **Project Name** - Descriptive title
   - **Project Type** - Category
   - **Start Date** - Project kickoff
   - **End Date** - Deadline
   - **Budget** - Financial allocation
   - **Project Manager** - Lead person
5. Define project scope
6. Click **Save**

## Tasks and Subtasks

Break down work into manageable pieces:

### Creating Tasks

Add tasks to your project:

1. Open the project
2. Click **Add Task**
3. Enter task details:
   - Task name
   - Description
   - Assigned to
   - Due date
   - Estimated hours
   - Priority
4. Set dependencies
5. Click **Save**

### Task Properties

Configure task attributes:

- **Status** - Not Started, In Progress, Completed, Blocked
- **Priority** - Low, Medium, High, Critical
- **Type** - Planning, Design, Build, Test, Deploy
- **Billable** - Billable or non-billable time
- **Dependencies** - Required predecessor tasks

### Subtasks

Break complex tasks into smaller pieces:

- Create unlimited subtask levels
- Track completion percentage
- Roll up time estimates
- Maintain context
- Filter and organize

## Dependencies

Manage task relationships:

### Dependency Types

- **Finish-to-Start** - Task B starts after Task A finishes
- **Start-to-Start** - Tasks start simultaneously
- **Finish-to-Finish** - Tasks finish simultaneously
- **Start-to-Finish** - Task B finishes when Task A starts

### Critical Path

Identify the critical path:

- Longest sequence of dependent tasks
- Tasks that cannot be delayed
- Project completion date
- Schedule optimization opportunities
