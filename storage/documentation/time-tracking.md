# Time Tracking

Track time entries, breaks, timesheets, and overtime. Learn how to log time accurately, manage timesheets, track breaks, and ensure proper billing for your work.

## Understanding Time Tracking

Accurate time tracking is essential for:

- Client billing and invoicing
- Project cost tracking
- Team productivity analysis
- Contract hour management
- Profitability assessment
- Payroll and overtime calculation

---

## Tracking Time

Multiple ways to track time in Nestogy:

### Real-Time Timer

Track time as you work:

1. Open a ticket or project task
2. Click **Start Timer**
3. Work on the task
4. Click **Stop Timer** when done
5. Review and save time entry

### Manual Time Entry

Enter time after the work is completed:

1. Navigate to Time Tracking
2. Click **Add Time Entry**
3. Select ticket or project
4. Enter start and end times (or duration)
5. Add description
6. Mark as billable/non-billable
7. Save entry

### Timer Widget

Quick access timer in navigation bar:

- Start/stop from anywhere
- Switch between tickets
- View elapsed time
- Pause and resume
- Quick notes

---

## Time Entry Properties

Configure time entry details:

### Basic Information

- **Date** - When work was performed
- **Start Time** - Beginning of work
- **End Time** - End of work
- **Duration** - Total time (auto-calculated or manual)
- **Description** - What was done

### Association

Link time to work items:

- Client
- Ticket number
- Project and task
- Contract (for bucket hours)
- Activity type

### Billing

Control how time is billed:

- **Billable** - Will be invoiced to client
- **Non-Billable** - Internal time, not charged
- **Billing Rate** - Hourly rate (from contract or default)
- **Total Amount** - Rate × duration

## Break Management

Track breaks and non-working time:

### Automatic Break Deduction

Configure automatic breaks:

- Settings → Time Tracking → Breaks
- Set break durations:
  - 15 minutes for shifts 4-6 hours
  - 30 minutes for shifts 6-8 hours
  - 1 hour for shifts over 8 hours
- Auto-deduct from time entries
- Compliance with labor laws

### Manual Break Tracking

Log breaks during the workday:

1. Click **Take Break** when starting break
2. Timer pauses automatically
3. Click **End Break** when returning
4. Break time excluded from billable time
5. Break durations logged separately

### Break Reports

Analyze break patterns:

- Break duration by team member
- Break frequency
- Unpaid vs. paid breaks
- Compliance tracking

---

## Timesheets

Weekly time summaries:

### Viewing Timesheets

Access timesheet view:

1. Navigate to Time Tracking → Timesheets
2. Select week
3. View time entries by day
4. See totals by client/project
5. Review billable vs. non-billable

### Timesheet Submission

Submit for approval:

1. Review all time entries
2. Add missing entries
3. Correct any errors
4. Click **Submit Timesheet**
5. Await manager approval

### Timesheet Approval

Managers approve team timesheets:

1. Review submitted timesheets
2. Check for errors or issues
3. Request corrections if needed
4. Approve or reject
5. Approved time ready for invoicing

## Billable vs. Non-Billable Time

Manage what gets invoiced:

### Billable Time

- Ticket resolution work

- Client meetings
- Client-requested work

### Non-Billable Time

Internal time not charged:

- Internal meetings
- Training
- Administrative tasks

### Billing Rate Override

Adjust rates per entry:

- Use contract rate (default)
- Custom rate for specific work
- Discounted rate
- Premium rate (after hours, emergency)
- Fixed price allocation

## Best Practices

Tips for accurate time tracking:

### Daily Tracking

- Track time same day
- Start timers immediately
- Stop timers promptly
- Review before submitting
- Be specific in descriptions

### Accuracy

- Round appropriately
- Include travel time policies
- Track breaks properly
- Document interruptions
- Be honest and ethical

### Efficiency

- Use timers, not memory
- Track as you work
- Batch similar tasks
- Minimize non-billable time
- Optimize workflows

### Communication

- Clear time descriptions
- Document what was done
- Note any issues
- Flag unusual entries
- Respond to approval feedback
